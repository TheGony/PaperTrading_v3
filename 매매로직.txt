1. 전략 개요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

두 가지 전략이 독립적으로 동시 운영됩니다.

  ORB (Opening Range Breakout)
    - 갭상승 종목 대상, 09:05~09:30 진입 윈도우
    - 09:00~09:04 봉으로 Range 확립 후 상단 돌파 시 매수
    - 빠른 청산 중심, 수익 반납 최소화

  MOMENTUM
    - 상승속도·수급응집도·거래대금·RSI·급증률 기반 장중 모멘텀 종목
    - 09:10 이후 ~ 15:20 진입 허용 (orb_ready 이후에만)
    - 추세 유지 중심, 눌림 허용

3-A. ORB 후보 선정 (orb_selector.py)
────────────────────────────────────────

  사용 API (2개 병렬 조회):
    ka10032  거래대금 상위 30종목       → 유동성 풀 (주)
    ka10030  당일 거래량 상위 30종목    → 유동성 풀 (주) + 거래량/거래대금 보조 맵

  풀 구성:
    ka10032 + ka10030 병합 (중복 시 첫 등장 우선), ETF/ETN/스팩 등 제외

  캔들 필터 (fn_ka10080_full, 20봉 조회):
    09:00~09:10 봉 추출 후:
    · 갭 계산: (당일 시가 - 전일 종가) / 전일 종가 × 100
    · 갭 -1% 미만 종목 제외
    · 갭 -1~0% 종목: 현재가 ≤ 시가이면 제외 (갭 회복 미확인)
    · 시가 대비 현재가 -2% 이상 눌린 종목 제외 (cur_prc < day_open × 0.98)

  1분봉 세부 지표 계산:
    · latest_high       : 최신봉 고가
    · upper_tail_ratio  : (최신봉 고가 - 현재가) / 최신봉 고가
    · bearish_count     : 09:00~09:10 음봉 개수
    · max_high_10       : 09:00~09:10 구간 최고가
    · price_position    : 현재가 / max_high_10  (1.0에 가까울수록 고점 유지)
    · consec_bull       : 최근 3봉 모두 양봉 여부
    · vol_3m            : 최근 3분 거래량 합계

  소프트 패널티 (스코어 감산):
    · 갭 범위 1.5~8% 벗어남              → +0.2
    · 윗꼬리 비율 > 5%                   → +0.2
    · 음봉 개수 ≥ 4                      → +0.4
    · 음봉 개수 ≥ 2 (4 미만)             → +0.2
    · price_position < 0.92              → +0.3  (10분 고점 대비 8% 이상 밀림)
    · price_position < 0.95 (0.92 이상)  → +0.15
    · consec_bull = True                 → -0.1  (연속 양봉 보너스)

  스코어 공식:
    score = log(trde_prica) / log(max_trde) × 0.50   거래대금 (log 정규화)
          + log(trde_qty)   / log(max_vol)  × 0.20   당일 거래량 (log 정규화)
          + flu_rt          / max_flu       × 0.30   등락률
          - penalty

  orb_data 선제 구축:
    · 선정 완료 후 09:00~09:04 봉으로 orb_high, orb_low, gap_up, day_open을
      self.orb_data[stk_cd]에 미리 저장 → _try_orb_entry의 중복 API 호출 방지

  결과:
    최대 10종목 (ORB_CANDIDATES_MAX)
    5종목 미만 시 캔들 필터 미통과 종목을 거래대금 순으로 보충 (score = -0.5)

3-B. MOMENTUM 후보 선정 (mom_selector.py)
────────────────────────────────────────

  사용 API (병렬 조회):
    ka10023  거래량 급증 종목 50종목    (주 — _from_surge=True 태그, sdnin_rt 포함)
    ka10030  당일 거래량 상위 50종목    (보조 — ka10023 미포함 종목만 보완)
    ka10032  당일 거래대금 상위 50종목  (보조 — 위 두 소스 미포함 종목만 추가)
    ka90009  외인 순매수 종목 집합      (수급 판단)

  병합 방식:
    · ka10023 먼저 seen dict에 저장 (_from_surge=True)
    · ka10030은 중복 종목에 trde_qty·trde_amt 보완만, 신규 종목만 추가 (_from_surge=False)
    · ka10032는 위 두 소스에 없는 종목만 추가 (trde_prica → trde_amt 매핑)
    · ka10030/ka10032 단독 종목: sdnin_rt 없음 → 경과 시간 비례 프록시 추정
      (elapsed_mins / 390 × now_trde_qty / prev_trde_qty × 100)
    · ka10023 필터 완화: sdnin_rt 하한 제거, flu_rt > 0이면 통과

  1차 필터:
    · ETF/ETN/레버리지/인버스/스팩 등 키워드 제외
    · 등락률 ≥ 23% 과열 종목 하드 제외
    · 등락률 ≥ +0.3% AND 거래대금 ≥ 10억 (trde_amt ≥ 1000, 백만원 단위)
      → 조건 미충족 시 원본 pool 전체 사용 (fallback)

  차트 필터 및 동적 지표 계산 (fn_ka10080, 캐시 활용, Semaphore(4) 병렬):
    · RSI(14) 계산용 봉수 부족 시 해당 종목 제외
    · speed_5m : (현재가 - 5분 전 봉 종가) / 5분 전 봉 종가 × 100 (%)
    · vol_power: 수급 응집도 — Hybrid Volume Power (최근 5봉)
                 캔들 위치(MFI 근사) × 방향 가중 거래량 조합
                 상승봉 거래량 ×1.2 / 하락봉 거래량 ×0.8 / 보합 ×1.0
                 Σ(price_pos × adj_vol) / Σ(adj_vol) × 100
                 0 = 저점 마감·하락압력 / 50 = 중립 / 100 = 고점 마감·매수 우위

  스코어 공식:
    score = (speed_5m - min_speed) / speed_range   × 0.35   상승속도 (min-max 정규화)
          + vol_power / max_vol_power               × 0.25   수급 응집도
          + log(trde_amt) / log(max_amt)            × 0.20   거래대금 (log 정규화)
          + rsi_val / 100                           × 0.10   RSI(14)
          + sdnin_rt / max_sdnin                    × 0.10   거래량 급증 비율 (ka10023)

    과열 페널티: 등락률 > +20% 시 score × 0.8 (≥ 23%는 상위 필터에서 하드 제외)

  결과:
    · 현재 시점 점수 기준 상위 stock_count 종목 선정 (설정값, 기본 7)
    · 부족 시 거래대금+등락률 기준 fallback 보충

    rank (선정순위 1~N), rsi

  갱신 주기:
    · 초기 선정: 09:10 (ORB 2차 갱신 완료 직후)
    · 주기 갱신: 매 5분 (300초)
    · 즉시 보충: 당일 손실 2회 종목 제거 시 다음 루프에서 즉시 갱신

4-A. ORB 매수 (entry.py _try_orb_entry)
────────────────────────────────────────

  실행 조건 (루프 레벨):
    · orb_ready = True (09:03 2차 선정 완료 이후)
    · 현재 시간 09:05 ≤ T ≤ 09:30

  시간 가드:
    · 현재 시간 < 09:05 이면 즉시 진입 취소 (Range 미확립 구간)

  ORB Range 확립 (최초 1회):
    · _get_orb_candidates에서 선제 구축한 self.orb_data 우선 사용
      (orb_candle_n ≥ 5인 경우에만 캐시 신뢰)
    · 캐시 없거나 신뢰 불가 시: fn_ka10080_full 조회 (09:00~09:04 봉 5개 이상 필요)
    · orb_high = 09:00~09:04 구간 최고가
    · orb_low  = 09:00~09:04 구간 최저가
    · gap_up   = (09:00봉 시가 > 전일 종가) 여부
    · 5봉 미달 시 진입 취소

  갭 회복 재확인:
    · 후보 선정 시 갭이 -1~0% 구간이었던 종목:
      현재가 < 시가이면 진입 취소 (갭 회복이 소멸된 경우)

  price_position 재확인:
    · 선정 당시 price_position < 0.90이면 진입 취소
      (고점 대비 10% 이상 눌린 종목 — 모멘텀 소진 가능성)

  진입 필터 (순서대로 평가):
    1. 갭상승 (gap_up = True) 필수
    2. 현재가 > orb_high (돌파 발생)
    3. 현재가 ≤ orb_high × 1.01 (추격매수 방지)
    4. 현재 거래량 ≥ 직전봉 거래량 × 1.5
    5. RSI(14) 50 이상 75 이하
    6. ORB 누적 매수 횟수 < orb_max_count (기본값 5)
    7. 쿨다운: 동일 종목 매도 후 20분 경과

  돌파 유지 확인 (_confirm_breakout):
    · 3초 동안 0.5초 간격으로 현재가(fn_ka10001) 재조회
    · 이탈 시 즉시 진입 취소 (False 반환)

  손절 기준 사전 계산:
    · orb_stop_pct = max((orb_low / 진입가 - 1) × 100, -2.0)
    · ORB 저점 이탈 또는 -2.0% 중 타이트한 쪽 (더 높은 가격)
    · entry_snapshot에 저장 → _profit_check_loop에서 사용

4-B. MOMENTUM 매수 (entry.py _check_charts_and_trade)
────────────────────────────────────────

  실행 조건 (루프 레벨):
    · 매 5초 실행
    · orb_ready = True (09:03 이후) 이후 진입 조건 평가 시작
    · is_entry_allowed() = True (15:20 이전)
    · selected_stocks 내 종목 AND 미보유 AND 미진입 대기 중

  동일봉 스킵:
    · 직전 체크와 동일 cntr_tm(체결시간)이면 진입 조건 재평가 생략
    · → 1분봉이 갱신된 경우에만 진입 조건 평가

  진입 필터 (순서대로 평가):
    1. 현재가 > breakout_high (직전 5봉 최고가 돌파)
    2. 현재가 ≤ breakout_high × 1.015 (추격매수 방지 +1.5%)
    3. 쿨다운: 동일 종목 매도 후 20분 경과
    4. RSI(14) 45 이상 70 이하
    5. 당일 동일 종목 손실 2회 미만
    6. 등락률 ≤ 23% (과열 제외, meta 기준)
    7. 거래량 필터: 현재봉 거래량 > 직전 5봉 평균 × 1.2
    8. VWAP 기관수급 필터 (_institutional_filter):
       · 현재가 > VWAP                              (VWAP 상단 진입)
       · (현재가 - VWAP) / VWAP ≤ 3%               (VWAP 대비 과열 제외)
       · 최근 5봉 거래대금 > 이전 5봉 거래대금 × 1.2 (실질 매수세 확인)
       VWAP 계산: TP = (고가 + 저가 + 종가) / 3 (실제 low_pric 사용), 오래된 봉부터 누적

  돌파 유지 확인 (_confirm_breakout):
    · 3초 동안 0.5초 간격으로 현재가(fn_ka10001) 재조회
    · 이탈 시 즉시 진입 취소

  entry_snapshot 저장 항목:
    entry_price, entry_rsi, entry_flu_rt, entry_vol_ratio, entry_score,
    is_foreign, kospi_flu, kosdaq_flu, strategy, confirm_secs,
    breakout_strength, chase_pct, entry_trde_amt, entry_trde_amt_rank,
    entry_time_min, entry_rank, recent_5bar_vol, market_state,
    selection_reason, high_pct, entry_vwap, vwap_gap_pct

  1분/3분 수익률 스냅샷 (_record_price_snapshots):
    · 매수 성공 직후 백그라운드 태스크 시작
    · +60초: fn_ka10001로 현재가 조회 → pl_1m 기록
    · +180초: fn_ka10001로 현재가 조회 → pl_3m 기록
    · 매도 완료(entry_snapshot pop) 후 자동 종료

  총자산 계산:
    · fn_kt00004(보유 종목) + fn_kt00001(예수금) 조회
    · 총자산 = 예탁자산평가액 + 주식평가금액
    · kt00004 실패 시: kt00002(추정예탁자산) 또는 last_known_assets 캐시 사용

  매수 금액 = 총자산 × (buy_ratio / 100)
  매수 수량 = int(매수 금액 / 현재가)  ← 수량 0이면 취소

5. [3단계] 매도 로직
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

────────────────────────────────────────
5-A. 데드크로스 청산 (entry.py, 매 5초)
────────────────────────────────────────

  대상: MOMENTUM 보유 종목
  조건: MA(chart_short) < MA(chart_long) [데드크로스]
        AND RSI(14) < 45
  → _sell_stock() 즉시 호출


────────────────────────────────────────
5-B. 손익 기반 청산 (exit.py, 매 1초)
────────────────────────────────────────

  [MFE / MAE 실시간 갱신]
    peak_profit[stk_cd] = max(peak, pl_rt)   ← MFE
    min_profit[stk_cd]  = min(min,  pl_rt)   ← MAE

  [전략 분기]
    entry_snapshot.strategy == 'ORB'       → ORB 청산 로직
    entry_snapshot.strategy == 'MOMENTUM'  → MOMENTUM 청산 로직

  ──────────────────────────────────────
  ORB 청산 우선순위 (위일수록 먼저 평가)
  ──────────────────────────────────────
  순위  조건                                           유형        발동
  1     진입 후 2분 이내 AND pl_rt < -1.2%             hard stop   즉시 매도
  2     pl_rt ≤ orb_stop_pct                          hard stop   즉시 매도
        (orb_stop_pct = max(ORB저점기준%, -2.0%))
  3     peak ≥ 1.5% 도달 후 pl_rt < 0%                soft stop   2회 연속 확인 후 매도
  4     pl_rt ≤ peak - orb_trailing(peak)             soft stop   2회 연속 확인 후 매도
        orb_trailing(peak) = clamp(1.0 + peak × 0.12,  1.0, 1.8)
        예) peak=2% → gap=1.24%  peak=5% → gap=1.60%  peak=7% → gap=1.80%(상한)
  5     pl_rt ≤ -2.0%                                  hard stop   즉시 매도 (최후 안전망)

  ──────────────────────────────────────
  MOMENTUM 청산 우선순위
  ──────────────────────────────────────
  순위  조건                                           유형        발동
  1     진입 후 2분 이내 AND pl_rt < -1.2%             hard stop   즉시 매도
  2     pl_rt ≤ peak - momentum_trailing(peak, vol)   soft stop   2회 연속 확인 후 매도
        momentum_trailing = clamp(2.0 + log(1+peak)×0.6 + vol×0.5,  2.0, 3.2)
        vol = market_volatility (KOSPI×0.4 + KOSDAQ×0.6 장중진폭%)
        예) peak=2%, vol=0.0 → gap=2.54%
            peak=5%, vol=1.0 → gap=3.20%(상한)
  3     pl_rt ≤ -3.0%                                  hard stop   즉시 매도 (최후 안전망)

  ──────────────────────────────────────
  휩쏘 방지 (_sell_signal_count)
  ──────────────────────────────────────
  · hard stop: 즉시 매도, signal_count 초기화
  · soft stop: 연속 2회 조건 충족 시 매도
               1회 충족 → count=1, 로그 기록, 다음 루프 대기
               2회 충족 → count=2, 매도 실행
  · 조건 미충족 시: count 즉시 0 (초기화)

  ──────────────────────────────────────
  매도 실패 처리
  ──────────────────────────────────────
  · API result ≠ 0: 상태 전체 유지 (entry_snapshot, peak_profit, signal_count)
  · 다음 1초 루프에서 자동 재평가 → 조건 충족 시 재시도

6. Market Regime (시장 체제 판단)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  목적:
    종목 단위 trailing 파라미터에 시장 변동성을 반영한다.
    Regime은 매수/매도 조건 판단에 직접 개입하지 않는다.

  갱신 주기:
    매 5분 (_regime_loop 백그라운드 태스크)

  데이터 소스:
    ka20001 (업종현재가요청) — 당일 OHLC + 등락률
      KOSPI  (mrkt_tp='0', inds_cd='001')
      KOSDAQ (mrkt_tp='1', inds_cd='101')

  계산:
    kospi_vol   = (KOSPI  high - low) / cur_prc × 100   장중 진폭(%)
    kosdaq_vol  = (KOSDAQ high - low) / cur_prc × 100

    kospi_tr    = |KOSPI  flu_rt|    전일 대비 변동 절댓값(%)
    kosdaq_tr   = |KOSDAQ flu_rt|

    market_volatility = kospi_vol × 0.4 + kosdaq_vol × 0.6
    market_trend      = kospi_tr  × 0.4 + kosdaq_tr  × 0.6

  Regime 분류:
    volatile_market  : market_volatility > 2.5   (장중 진폭 과대)
    trend_strong     : market_trend      > 0.8   (지수 뚜렷하게 움직임)
    sideways         : market_trend      < 0.3   (지수 거의 보합)
    normal           : 나머지

  Regime 사용:
    · MOMENTUM trailing: momentum_trailing(peak, market_volatility)
      → vol 값이 클수록 trailing gap 확대 (눌림 허용폭 증가)
    · ORB trailing: 시장 volatility 미사용 (고정 연속 함수)

  상태 변수:
    self.market_volatility  float   현재 시장 변동성
    self.market_regime      str     'normal' / 'sideways' / 'trend_strong' / 'volatile_market'
